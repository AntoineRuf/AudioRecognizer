void gracenote::video::GnVideoOptions::QueryNoCache ( bool  bNoCache)
throw (GnError
)

Indicates whether a response is not automatically stored in the cache.

Parameters
bNoCacheSet boolean to enable caching

Remarks:

Set this option to True to retrieve query data from a call to Gracenote Service; doing this ensures retrieving the most recent available data. Set this option to False to retrieve query data that already exists in the lookup cache. When no data exists in the cache, VideoID or Video Explore automatically calls Gracenote Service with the query request.

void gracenote::video::GnVideoOptions::ResultCount ( gnsdk_uint32_t  resultCount)
throw (GnError
)

Set number of results to be returned

Parameters
resultCount[in] Maximum number of results a response can return.

Remarks:

This option is useful for paging through results.

Note:

Gracenote Service limits the range size for some queries. If you specify a range size greater than the limit, the results are constrained. Additionally, neither Gracenote Service nor GNSDK returns an error about this behavior.

Important:

The number of results actually returned for a query may not equal the number of results initially requested. To accurately iterate through results, always check the range start, end, and total values and the responses returned by Gracenote Service for the query (or queries). Ensure that you are incrementing by using the actual last range end value plus one (range_end+1), and not by simply using the initial requested value.

const GnListElement &  listElement,
bool  bInclude

Sets a filter for a Video Explore query handle, using a list value.

Parameters
bInclude[in] Set Boolean value to True to include filter key; set to False to exclude the filter
listElementFilterType[in] One of the GnVideoListElementFilterType filter types
listElement[in] A list element handle used to populate the filter value. The list element must be from a list that corresponds to the filter name.

Remarks:

Use this function with the GNSDK Manager Lists functionality to apply filters on and improve the relevance for specific Video Explore search results. Your application must have Video Explore licensing to use this function.

Note:

The system disregards filters when performing non-text related video queries, and filtering by list elements is restricted to performing a Works search using the following Filter Values:

  • kListElementFilterGenre
  • kListElementFilterProductionType
  • kListElementFilterSerialType
  • kListElementFilterOrigin

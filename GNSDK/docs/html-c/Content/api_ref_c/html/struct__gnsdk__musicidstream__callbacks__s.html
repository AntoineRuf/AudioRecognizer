Detailed Description

Structure of callback functions for status updates as various MusicID-Stream operations are performed.

Remarks:

Note:

Ensure that callback functions that access the same global variable(s) use serialized access methods to the shared variable(s).

Field Documentation

gnsdk_musicidstream_completed_with_error_callback_fn _gnsdk_musicidstream_callbacks_s::callback_error

MusicID-Stream audio returns an error during identification for the given MusicID-Stream channel handle. Due to the error, no result will be provided.

gnsdk_musicidstream_identifying_status_callback_fn _gnsdk_musicidstream_callbacks_s::callback_identifying_status

Callback function during audio processing when a significant event occurs such as large classification change or transition.

gnsdk_musicidstream_processing_status_callback_fn _gnsdk_musicidstream_callbacks_s::callback_processing_status

Experimental: An application should only provide this callback if it is advised by Gracenote Global Services and Support representative. Contact your Gracenote Global Services and Support representative with any questions about this enhanced functionality.

gnsdk_musicidstream_result_available_callback_fn _gnsdk_musicidstream_callbacks_s::callback_result_available

Result has been determined by MusicID-Stream for a given audio stream within a MusicID-Stream channel.

gnsdk_status_callback_fn _gnsdk_musicidstream_callbacks_s::callback_status

Callback function during identification when an online query is performed.

Monitoring Memory Usage

GNSDK for Desktop contains the following APIs that enable you to have better visibility and control over memory usage. The first enables you to track and measure both heap usage and number of allocations for your particular deployment. The set of metrics available to the developer using this API will grow over time with future SDK updates.

gnsdk_manager_internals();

GNSDK for Desktop also has the following API that can be set to call back to the application when GNSDK for Desktop reaches a certain level of memory usage. This callback is informational only. The application will have to decide for itself the right course of action, if it gets this callback.

gnsdk_manager_memory_warn();
